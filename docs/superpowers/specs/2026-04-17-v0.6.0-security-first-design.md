# Security-First Release Strategy

## Overview

This design documents the Security-First approach (Approach C) for completing v0.6.0-v0.7.0 features. The strategy prioritizes security hardening and observability before shipping new features, ensuring a production-ready foundation.

**Timeline:** 6-7 weeks solo (30-35 working days)

## Phases

### Phase 1: Finish v0.6.0 (Week 1)

#### S10.5 - PR Review Cleanup
- **Goal:** Merge remaining PRs to main
- **Scope:** Fix 4 review comments on PR #8-#10
  - `daemon/security.py:18` - Fix overly broad pattern
  - `daemon/main.py:981` - Fix sanitize_text mangling
  - `daemon/security.py:35` - Fix delimiter sanitization
  - `daemon/sync_watcher.py:434` - Deduplicate variable
- **Deliverable:** PRs merged, v0.6.0 stable foundation

#### S17 - Security Hardening
- **Goal:** Production-grade security controls
- **Scope:**
  - `daemon/validation.py` - Path traversal, slug, text validation
  - Rate limiting via slowapi (cognify 10/min, promote 5/min, ingest 3/min, search 30/min)
  - Security headers (X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, HSTS)
  - Sensitive operation audit logging
- **Deliverable:** v0.6.1 production release

### Phase 2: Production Ready (Week 2)

#### S16 - Observability
- **Goal:** Monitoring and diagnostics
- **Scope:**
  - `/health/detailed` - PostgreSQL, Weaviate, disk, memory checks
  - `/metrics` - Prometheus metrics (search_counter, search_duration, cognify_counter)
  - `/dashboard` - HTML KPI view
- **Deliverable:** Production monitoring stack

#### S20 - Batch Optimization
- **Goal:** Performance improvements
- **Scope:**
  - Optimize `sync_file` and `_sync_canvas` batch methods
  - `embed_batch` + `batch_upsert` for multi-chunk files
  - Regression tests for latency
- **Deliverable:** 78% performance improvement

### Phase 3: New Features (Weeks 3-4)

#### S18 - SQLite Lite
- **Goal:** Embedded deployments
- **Scope:**
  - `daemon/lite_store.py` with PG schema parity
  - CLI `--lite` flag and `VAULT_MEMORY_LITE_MODE` env var
  - `LiteEmbedder` with local sentence-transformers fallback
  - `migrate-to-lite` command
- **Deliverable:** Dockerless deployment option

#### S19 - Obsidian Plugin
- **Goal:** Native Obsidian integration
- **Scope:**
  - `obsidian-plugin/` (TypeScript, esbuild)
  - Search modal (Ctrl+Shift+F) with hybrid/topology/vector modes
  - Ingest command for atomic source→knowledge workflow
  - Graph view with D3 visualization
  - Auto-sync engine
  - Settings tab (daemon URL, API key, preferences)
- **Deliverable:** Obsidian plugin release

## Dependencies

```
S10.5 (PR cleanup) → S17 (Security) → S16 (Observability)
                            ↓
                         S20 (Batch optimization)
                            ↓
                         S18 (SQLite Lite)
                            ↓
                         S19 (Obsidian Plugin)
```

**Key constraints:**
- S19 requires S18 complete (backend dependency)
- S16 and S17 can parallelize if needed
- All features require S10.5 merged first

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Security bugs after hardening | Low | Extensive testing, audit logging |
| Batch optimization regressions | Medium | Regression tests, gradual rollout |
| SQLite compatibility issues | Medium | PG schema parity, migration tool |
| Plugin browser support | Low | Test all major browsers |

## Success Criteria

1. **v0.6.1 security release** - All security headers, rate limits enforced
2. **Performance verified** - Batch optimization shows 78% improvement
3. **SQLite works** - Embedded deployment runs without Docker
4. **Plugin loads** - Obsidian plugin integrates search, graph, sync
